# Kamigotchi 自动化脚本 · 安装使用说明

> **Kamigotchi**（kamigotchi.io，链上宠物采集游戏）挂机自动化脚本套件 —— 自动部署 / 停采 / 喂食 / 复活 / 合成 / 拾荒 / 杀手监控。
> **仅支持 Chrome 浏览器 + Tampermonkey（篡改猴）。** 装一次，以后自动更新。

> ⚠️ **免责声明**：本套脚本是作者**自用**工具，按「现状（as-is）」公开分享。它会代你**发送真实链上交易、消耗 gas，并自动部署 / 停采 / 喂食 / 升级 / 重置技能**你的 Kamigotchi、**自动合成物品**。**使用即代表你自担全部风险**——请务必先**自行阅读代码、在小号或少量 Kamigotchi 上充分测试、确认行为符合预期**后，再用于主力账户。作者**不保证适用性、不对任何资产损失负责、不承担技术支持义务**。

## 脚本组成（装 4 个，平时常驻 3 个）

| 脚本 | 作用 | 何时运行 |
|---|---|---|
| **核心脚本 (core)** | 主流程：部署 / 停采 / 喂食 / 复活 / XP 药水 / 拾荒 / gas 统计 | **常驻**（一直开着）|
| **辅助脚本 (helper)** | 升级技能 / 自动合成 / 清算线计算 / 健康看板 | **常驻** |
| **轻量杀手监控 (killer monitor)** | 轮询杀手位置，检测到威胁自动联动紧急停采 | **常驻** |
| **精简数据库 (database)** | 扫描账户全部 Kamigotchi，构建本地数据库（编号 / ID / 清算线 LT 等） | **只首次建库跑一次，之后停用**；换账户或数据异常时再开一次重建 |

> 平时常驻运行的是 **3 个**：核心 + 辅助 + 杀手监控。**精简数据库不是常驻脚本**——首次建好本地库后就停用。**之后新增的 Kamigotchi 会被核心脚本自动补进数据库**（增量自愈，无需任何手动操作）；想手动全量重建也可以（重新启用数据库脚本刷新一次，或控制台执行 `rebuildKamiCoreDb()`）。

## 功能一览

**清算线 & 威胁分析**

1. **精确清算线**：根据当前**全网针对不同 body 类型的最厉害杀手**，自动计算每只 Kamigotchi 的清算线（LT）；
2. **每周自动扫描一次全网杀手**，更新杀手信息，清算线随之刷新；
3. 自动把清算线**插入游戏前端界面**显示，**标红危险的 Kamigotchi**；
4. **自动识别你账户里哪些 Kamigotchi 有「做杀手」的潜质**（攻击向数据突出的个体）。

**采集主循环（API 批量、尽量省 gas）**

5. **自动部署**空闲（RESTING）Kamigotchi 去采集点，凑批发送摊薄 gas；
6. **自动停采**（HP 触及停采线时），批量发送；有杀手威胁时紧急停采；
7. **自动喂食物**（低血量按 HP 缺口智能选食物）；
8. **自动复活**死亡的 Kamigotchi（需背包有复活丝带）；
9. **自动升级高清算线的 Kamigotchi**（清算线越高越危险，优先升级降险）；
10. **自动喂经验药水**（只喂清算线 **> 70%** 的高危 Kamigotchi，加速升级降低清算线）；
11. **自动合成常用物品**（如 Greater XP Potion，步长充足时）；
12. **自动拾荒**（拾荒次数 rolls 攒到 **1000** 自动领取一次，一笔 tx 领全部奖励，省 gas）。

**杀手防护**

13. **轮询全网杀手位置**，威胁进入你所在/相邻地块时，自动切安全停采线 / 触发紧急停采（主人 >24h 不活跃的杀手视为低威胁，不误停）。

**一键工具 & 运维**

14. **一键停采全部 Kamigotchi**（方便整体转移地块采集）；
15. **一键停采少数派 Kamigotchi**（方便转移到其他账户）；
16. **每 ~45 分钟自动强制刷新页面一次**（有操作进行中会顺延，避免网页长时间挂起变卡）；
17. **自动下载保存运行日志**，方便复盘（⚠️ 首次需在 Chrome 地址栏手动点一次「**允许**下载多个文件」，否则只存不下载）；
18. **gas 消耗统计 / 余额耗尽预估 / 低余额充值预警**；
19. **代码健康看板**：自检各模块是否正常、异常标红；
20. **操作冷却预筛**：任何操作后约 180 秒链上冷却期内不发无效 tx，省 gas；
21. **前端卡死传感器**（纯日志）：检测网页同步是否冻死（用于排查"没停成/没喂上"）；
22. **启动版本检查**：每个脚本启动时自动对比 GitHub 最新版本，控制台提示「✅已是最新 / ⚠️请到篡改猴检查更新」（含该版发布时间与本机更新时间）。

## 关键参数（默认值 + 在哪调）

> 想改这些数值需要 fork 自己一份改（原理同文末「respec」那节——本地改会被自动更新覆盖）。

| 参数 | 默认值 | 说明 / 在哪改 |
|---|---|---|
| **贪婪模式停采线** | **5%** | 地块**一直没有杀手威胁**时，血量降到 5% 才停采（榨干采集周期，收益最高）；检测到杀手自动切回安全线。切换命令 `setKamiMode('greedy')` |
| **安全模式停采线** | **清算线 LT + 3%**（最高封顶约 80%）| 默认/有威胁时用；越高越保命但采集越短 |
| **经验药水喂食门槛** | **清算线 > 70%** | 只给高清算线的高危 Kamigotchi 喂 XP 药水（升级降清算线）。改：核心脚本搜 `XP_POTION_LT_THRESHOLD = 70` |
| **部署凑批门槛** | **6 只** | 候选 RESTING **< 6 只时跳过部署本轮**（凑批省 gas，适合 kami 很多的账户）。**测试/小号想改**：核心脚本搜 `MIN_DEPLOY_BATCH = 6`，调小（如 1~2）|
| **停采凑批门槛** | **6 只** | 根据 gas 消耗实测：**批量 ≥6 只起单只均摊 gas 明显下降**（N=6 比单只省约 37%），故凑够 6 只 urgent 才批量停采（有危险 kami 不等凑批、立即停）。改：核心脚本搜 `TARGET_BATCH_MIN = 6` |
| **主循环周期** | **10 分钟**（+随机）| 多久扫一轮账户 |
| **强制刷新周期** | **45 分钟**（+随机 0~5 分钟）| 核心脚本搜 `baseMinutes = 45`；有操作进行中会顺延（最多 3 次×3 分钟）|
| **拾荒领取门槛** | **1000 rolls** | 拾荒次数攒到 1000 才点一次 Scavenge，一笔 tx 领取全部奖励（省 gas）。改：核心脚本搜 `rolls < 1000` |
| **操作冷却** | **约 180 秒** | 任何操作后的链上冷却，脚本会预筛避开、不白发 tx |

> ⚠️ **本套脚本面向 kami 数量 > 7 的大账户设计**（凑批门槛 6）。**小账户 / 测试**：务必把 `MIN_DEPLOY_BATCH` 和 `TARGET_BATCH_MIN` 调小（如 1~2），否则可能长时间凑不满一批而**不部署 / 不停采**。

## 第 1 步：安装 Tampermonkey（篡改猴）

打开 Chrome 应用商店官方安装页，点 **「添加至 Chrome」**：

**https://chromewebstore.google.com/detail/tampermonkey/dhdgffkkebhmkfjojejmpbldmpobfkfo**

装完 Chrome 右上角会出现篡改猴图标（黑色方块小狗头）。

---

## 第 2 步：⚠️ 打开「允许用户脚本」（最关键，漏了脚本装了也不跑）

新版 Chrome 出于安全，默认**不允许**扩展运行用户脚本，必须手动开一下。**篡改猴装好后会弹一个红色横幅提示你**——照它说的做即可。两种情况（看你 Chrome 版本）：

### 情况 A：新版 Chrome —— 开「允许用户脚本」（推荐，更安全）
1. 地址栏输入 `chrome://extensions` 回车；
2. 找到 **Tampermonkey** → 点 **「详细信息 / Details」**；
3. 找到 **「允许用户脚本 / Allow user scripts」** 开关 → **打开**。

### 情况 B：老一点的 Chrome —— 开「开发者模式」
1. 地址栏输入 `chrome://extensions` 回车；
2. 右上角 **「开发者模式 / Developer mode」** 开关 → **打开**。

> 不确定哪种？**看篡改猴面板顶部的红色横幅**，它会明确写「请打开 Allow user scripts」或「请打开开发者模式」，点它给的链接直接跳到设置处。开好后横幅消失即成功。

## 第 3 步：安装 4 个 Kamigotchi 脚本（从 GitHub 一键装）

在 Chrome 地址栏**依次打开**下面 4 个链接，每个都会弹出篡改猴的**安装页**，点 **「安装 / Install」**：

1. **核心 (core)**：https://raw.githubusercontent.com/funcreator2030/kamigotchi-scripts/main/kamigotchi-core.user.js
2. **辅助 (helper)**：https://raw.githubusercontent.com/funcreator2030/kamigotchi-scripts/main/kamigotchi-helper.user.js
3. **精简数据库 (database)**：https://raw.githubusercontent.com/funcreator2030/kamigotchi-scripts/main/kamigotchi-database.user.js
4. **轻量杀手监控 (killer monitor)**：https://raw.githubusercontent.com/funcreator2030/kamigotchi-scripts/main/kamigotchi-killer-monitor.user.js

> 也可以直接进仓库首页挑：**https://github.com/funcreator2030/kamigotchi-scripts**

**4 个都装**（安装页看脚本名括号里的英文区分是哪个）。但注意——**平时常驻只开 3 个（核心 + 辅助 + 杀手监控）；精简数据库首次建好库后要停用**（见下一步）。

> ⚠️ **如果这台 Chrome 以前手动粘贴装过旧版**：先在篡改猴面板里**删掉旧的那几个**，再装上面的 GitHub 版。否则新旧两份同时跑，会重复发交易、乱套。

## 第 4 步：首次建库（精简数据库脚本跑一次，之后停用）

「精简数据库 (database)」负责扫描你账户的全部 Kamigotchi、构建本地数据库（每只的编号 / ID / 清算线 LT）。它**只在首次（或换账户 / 数据异常时）跑一次**，不是常驻。

1. 4 个脚本都启用的状态下，打开 Kamigotchi 游戏页 `https://*.kamigotchi.io`；
2. 数据库脚本会**自动开始建库**（约 1~2 分钟：先等约 40 秒，再自动展开 Party 面板扫描）；
3. 控制台出现 **`🎉 构建完成`** 并列出全库表格 = 建好了；
4. **在篡改猴面板里把「精简数据库 (database)」停用**（关掉它的开关）。之后日常新增的 Kamigotchi 由核心脚本自动增量补进库，不必再全量重扫。
   > 想全量重建库时（换登录账户、数据明显异常）：再启用数据库脚本、刷新页面即可；或在控制台执行 `rebuildKamiCoreDb()`。

## 第 5 步：验证是否装好

1. 打开 Kamigotchi 游戏页 `https://*.kamigotchi.io`；
2. 按 **F12** 打开控制台（Console）；
3. 看到类似 **`✅ Kamigotchi核心脚本-公开版 vX.X.X 已成功启动`** 就成功了（8 秒后还有一条「✅ [版本检查] …已是 GitHub 最新」）；
4. 篡改猴图标上**平时显示 3**（核心 + 辅助 + 杀手监控常驻；精简数据库已停用，只在建库时才是 4）。

## ⚠️ 夜间挂机：先关闭电脑自动睡眠

脚本运行在浏览器页面里，**电脑一旦睡眠，页面 JS 全部暂停**——停采/喂食都不会执行，Kamigotchi 可能被杀手清算。**晚上长时间挂机前必须关闭自动睡眠**（脚本启动时控制台也会提示这条 💤）：

- **Mac**：系统设置 → 能耗 → 「显示器关闭时防止自动进入睡眠」**打开**；
- **Windows**：设置 → 系统 → 电源 → 「使设备保持唤醒状态」选 **「永不」**。

2. **管理面板 → 已安装脚本**列表，悬停某脚本的**「最后更新」列**点「检查更新」（查单个）。

## ⚠️ 重要提醒：脚本会自动重置技能点（respec）

**辅助脚本 (helper) 会自动管理技能点**：它按内置的「标准采集 build」自动加点；一旦发现某只 Kamigotchi 点了**非标准技能**，会消耗 **1 瓶 Respec 药水**把它**全部技能点洗掉重置**，再按标准 build 重新加点。

- ✅ **杀手 Kami 自动跳过**（在杀手监控脚本配置过、或已进 `window.MY_KILLER_KAMIS` 的不会被重置）；
- ⚠️ 如果你有**采集 Kami 想保留自己的技能加点**、不想被重置 —— 需要自己改脚本（见下）。

### 「标准 build」具体是哪些技能（18 个，编号+名字）

作者面向「少被清算、采集更久」目标的加点方案——**生存（Guardian 树）12 个 + 收益（Harvester 树）6 个**。Kamigotchi 点了**这 18 个之外**的技能就会被判非标准、触发重置：

| 系别 | 编号 · 技能名 |
|---|---|
| **生存 Guardian · T1** | 311 Defensiveness · 312 Toughness · 313 Patience |
| **生存 Guardian · T2** | 321 Meticulous · 322 Vigor · 323 Armor |
| **生存 Guardian · T3** | 331 Anxiety（单点）|
| **生存 Guardian · T4** | 341 Flawless · 343 Shielding（不含 342）|
| **生存 Guardian · T5** | 352 Hefty · 353 Fortress（不含 351）|
| **生存 Guardian · T6** | 361 Neurosis（单点）|
| **收益 Harvester · T1** | 411 Acquisitiveness · 412 Mogging · 413 Greed |
| **收益 Harvester · T2** | 421 Wide Portfolio · 422 Side Hustles |
| **收益 Harvester · T3** | 431 Technical Analysis（单点）|

**自动加点顺序**（`SKILL_ORDER`，生存优先）：
`311 → 312 → 321 → 323 → 331 → 322 → 343 → 341 → 313 → 353 → 352 → 361 → 411 → 412 → 421 → 431 → 413 → 422`

> 这套 build 是作者的个人策略，不是唯一解。想换自己的方案：fork 后同步改 `STANDARD_SKILLS` / `SKILL_ORDER` / `SKILL_CAP` 三处（不同步会陷入"刚点完就被判非标准又重置"的循环）。

### 不想被重置怎么改（要 fork 自己一份）

脚本是**从本仓库自动更新**的，你直接改本地装的那份，**下次自动更新会把改动覆盖掉**。所以要让改动生效并保住，得 fork 自己一份、从你自己的 fork 安装：

1. 打开 **https://github.com/funcreator2030/kamigotchi-scripts** → 右上角 **Fork** 到你自己账号；
2. 在**你的 fork** 里编辑 `kamigotchi-helper.user.js`：
   - **豁免指定 Kami**（保留其自定义技能，最常用）：搜索 `RESET_WHITELIST`（约 629 行），把要豁免的 Kami 编号（游戏里名字旁 `#` 后的数字）填进去：
     ```js
     const RESET_WHITELIST = new Set([
       1234, 5678,   // 这几只不自动升级 / 不重置 / 不加点
     ]);
     ```
   - **或想换成你自己的加点方案**：同步改 `STANDARD_SKILLS` / `SKILL_ORDER` / `SKILL_CAP` 三处（脚本内有板块注释指引）；
   - 🔑 **关键一步**：把该文件头部的 `@downloadURL` / `@updateURL` 改成**你自己 fork 的 raw 地址**（否则篡改猴仍从原仓库拉更新、把你的改动覆盖）。想彻底不被更新，也可**直接删掉这两行**。
3. 从**你 fork 的 raw 地址**重新安装 `kamigotchi-helper.user.js`（替换原来那份）。

> 只需 fork + 改**辅助脚本这一个**；核心 / 数据库 / 杀手监控 3 个继续用原仓库的自动更新版即可。

## 常见问题

| 现象 | 原因 / 解决 |
|---|---|
| 脚本装了，控制台没有启动日志、图标不显示数字 | 十有八九是**第 2 步没做**（没开「允许用户脚本」/开发者模式）。回去开一下，刷新 Kamigotchi 页面。 |
| 打开 raw 链接没弹安装页，只显示一堆代码 | 篡改猴没装好或没启用；确认扩展已启用，再重开链接。 |
| 装完发现交易重复、行为异常 | 旧的粘贴版没删，和 GitHub 版并存了。删掉旧的那几个。 |
| 提示「此脚本更新源不受信任」 | 首次从 raw 装属正常提示，确认链接是本仓库地址后点信任/安装。 |
| 想临时停某个脚本 | 篡改猴面板里把对应脚本开关关掉即可。 |
